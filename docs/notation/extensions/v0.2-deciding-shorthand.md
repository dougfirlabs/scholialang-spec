# Scholia v0.2 extension proposal - short-form `<Deciding>`

**Status:** accepted for v0.2
**Target version:** Scholia v0.2
**Impact:** parser sugar only; no new atom.

## 1. Motivation

Simple two-option decisions are verbose in v0.1. The full Deciding/Branch/Finding form is still canonical, but agents should be able to write the common case compactly.

## 2. Proposed syntax

```xml
<Deciding id="Decision_01" options="A, B" chose="A">
  A is cheaper and satisfies the required constraint.
</Deciding>
```

The parser desugars this to a normal `<Deciding>` with `options`, one `<Branch>` per option, and one `<Finding>` declaring the chosen option plus rationale. The AST has no short-form flag.

## 3. Canonical output

Serializers and renderers always emit the long form. Short form is input syntax only.

## 4. Errors

`chose` must exactly match one listed option after whitespace trimming. Malformed short form fails at parse time with a clear error.

## 5. Backward compatibility

Long-form v0.1 Deciding remains valid and unchanged.
