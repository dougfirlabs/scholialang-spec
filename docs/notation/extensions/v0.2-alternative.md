# Scholia v0.2 extension proposal - `<Alternative>` atom

**Status:** accepted for v0.2
**Target version:** Scholia v0.2
**Impact:** adds one control atom constrained to `<Deciding>` children.

## 1. Motivation

v0.1 can record a chosen branch, but rejected options often remain prose inside a Deciding block. Monitors need a stable place to inspect options that were considered and rejected.

## 2. Proposed atom

```xml
<Deciding id="Decision_01">
  <Alternative label="defer" rejected_because="Blocks the required Goal." />
  <Alternative label="ship now" rejected_because="Tests are not green." />
  <Branch of="Decision_01" label="fix_then_ship" />
  <Finding>Chose fix_then_ship.</Finding>
</Deciding>
```

`label` is the rejected option's short name. `rejected_because` is the rationale. Body content may carry longer prose when needed.

## 3. Structural rule

`<Alternative>` is only legal inside `<Deciding>`. This is a parse-time structural check, not a validity rule.

## 4. Backward compatibility

Existing Deciding blocks without Alternative children continue to parse and validate.

## 5. Alternatives considered

1. Extend `<Branch>` with `chosen="false"`. Rejected because Branch is a legal transition, while Alternative is explicitly rejected.
2. Keep alternatives in prose. Rejected because monitors cannot reliably enumerate them.
